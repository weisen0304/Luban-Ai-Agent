Deploy

这套官网已经适合直接发布到静态托管平台

当前站点全部由静态 HTML、CSS 和本地资源构成，天然适合部署到 GitHub Pages、 Cloudflare Pages 或任意静态文件服务器。

当前特性

零构建、零依赖、直接打开就能运行

  • 根目录页面可直接作为入口发布
  • 所有页面都使用相对路径
  • 资源已经统一收纳到 assets 目录

Publishing Steps

推荐发布方式

01

GitHub Pages

最适合当前仓库结构。直接选择 `main` 分支根目录作为发布源即可。

02

静态服务器

把整个目录上传到任意 Nginx、Apache 或对象存储静态站点即可运行。

03

后续演进

如果以后需要表单、接口或后台，再升级为前后端项目也完全兼容。

Release Checklist

发布前建议检查

内容

检查页面文案

确认首页、员工页、SKILL 页和详情页的文案都符合你想对外展示的口径。

资源

检查图片与路径

确保头像、Logo 和详情页链接都能正常访问，没有断链或丢图。

入口

检查导航闭环

确保首页、列表页、详情页和部署页之间都能互相返回，不形成死路。
