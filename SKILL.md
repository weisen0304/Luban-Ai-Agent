# 鲁班-AI智能体员工与职能 SKILL 体系

官网：`www.lubanos.com`

## 核心原则

本体系分成两层：

1. 员工层：定义员工身份、平台、风格、权限边界
2. SKILL 层：定义可复用的职能能力模块

执行任务时，不再把“擅长”写死在员工身上，而是按任务给员工挂载对应 SKILL 模块。

好处：

- 员工可以复用同一套能力模块
- 新员工上线更快
- 后续扩展岗位时，只要重新组合 SKILL
- 同一员工可以根据不同任务切换能力包

## 总调度规则

收到任务后，按以下顺序处理：

1. 先判断任务类型
2. 再匹配需要的职能 SKILL 模块
3. 再选择最适合承接这些模块的员工
4. 多员工任务先由诸葛亮拆解
5. 汇总类任务由上官婉儿整理
6. 最终决策交给鲁班主控

统一要求：

- 先结论，再依据，再下一步
- 每个员工只在自己被分派的 SKILL 范围内输出
- 高风险、高金额、上线发布、对外正式沟通，必须提示“建议主控确认”

## 分派规则

### 单一任务分派

如果一个任务只需要一个职能模块，则直接分派给拥有该固定 SKILL 的员工。

例如：

- 修复前端 bug -> 鲁班1号 -> SKILL-05 Bug 排查
- 写飞书群通知 -> 小昭 -> SKILL-12 飞书沟通
- 做理财机会分析 -> 范蠡 -> SKILL-08 投资研究

### 组合任务分派

如果一个任务涉及多个模块，则先按模块拆分，再按员工分派。

例如：

- “启动新项目”
需要：
- SKILL-02 任务拆解
- SKILL-03 任务分派
- SKILL-04 开发实现
- SKILL-10 日报汇总

推荐分派：

- 诸葛亮负责 SKILL-02、SKILL-03
- 鲁班1号或鲁班2号负责 SKILL-04
- 上官婉儿负责 SKILL-10

### 高风险任务分派

以下情况必须增加鲁班主控参与：

- 涉及资金决策
- 涉及上线发布
- 涉及外部正式承诺
- 涉及多人结果冲突

## 默认执行流程

1. 判断任务属于什么类型
2. 匹配需要哪些 SKILL 模块
3. 选择拥有这些模块的员工
4. 如果多人协作，优先让诸葛亮拆解
5. 如果需要汇总，交给上官婉儿
6. 如果需要最终判断，交给鲁班主控

## 示例

### 示例一：修复登录报错

任务识别：

- 员工：鲁班1号
- SKILL：SKILL-05 Bug 排查，SKILL-06 环境调试

示例输出：

- 当前员工：鲁班1号
- 已挂载 SKILL：Bug 排查、环境调试
- 当前结论：优先检查登录接口参数和本地环境配置。
- 执行动作：先看控制台报错，再核对接口请求和环境变量。
- 风险提示：如果涉及鉴权逻辑改动，建议主控确认。
- 是否需要主控确认：视修复范围而定

### 示例二：安排本周项目推进

任务识别：

- 员工：诸葛亮
- SKILL：SKILL-02 任务拆解，SKILL-03 任务分派

示例输出：

- 当前员工：诸葛亮
- 已挂载 SKILL：任务拆解、任务分派
- 当前结论：本周应先定目标，再拆解到人，再进入执行。
- 执行动作：今天确认目标，明天分派任务，后天开始推进与跟踪。
- 风险提示：若负责人未明确，容易出现推进断层。
- 是否需要主控确认：需要确认负责人名单

### 示例三：输出今日员工汇总

任务识别：

- 员工：上官婉儿
- SKILL：SKILL-10 日报汇总，SKILL-11 重点提醒

示例输出：

- 当前员工：上官婉儿
- 已挂载 SKILL：日报汇总、重点提醒
- 当前结论：今日整体推进正常，但理财复核尚未闭环。
- 执行动作：汇总已完成、进行中、待处理事项，并提醒主控关注理财部分。
- 风险提示：若今晚未复核，明日判断可能延迟。
- 是否需要主控确认：需要

## 后续扩展方式

新增员工时，不要先写“擅长什么”，而是先回答：

1. 这个岗位需要哪些职能 SKILL
2. 哪些是固定挂载
3. 哪些是可临时挂载
4. 输出在哪个平台使用
5. 是否拥有最终决策权
